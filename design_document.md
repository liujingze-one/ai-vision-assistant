# AI Vision Assistant 设计说明

## 一、用户故事

### 计划实现的用户故事

1. 作为用户，我希望 AI 能够看到我当前所处的环境，从而帮助我识别物体和理解场景。

2. 作为用户，我希望通过语音提问，而不是手动输入文字，从而提高交互效率。

3. 作为用户，我希望 AI 能够通过语音回复，从而获得更加自然的交流体验。

4. 作为用户，我希望能够连续提问，并查看历史对话记录。

### 最终实现情况

以上用户故事均已实现，具体功能包括：

* 摄像头实时预览
* 自动采集视频中的连续三帧画面
* Gemini 视觉问答
* 语音输入
* 自动发送问题
* AI 语音播报
* 多轮对话记录
* 页面界面优化

## 二、运营成本控制

### 想到的成本控制技巧

开发过程中考虑过两种方案：

#### 方案一：持续上传视频流

让系统持续上传视频内容并实时调用大模型进行分析。

优点：

* 能获取完整视频信息

缺点：

* API 调用频繁
* Token 消耗较高
* 运营成本较高

#### 方案二：关键帧采样

仅在用户提问时，从视频流中截取少量关键画面发送给模型。

优点：

* 大幅减少图像上传量
* 降低 Token 消耗
* 降低 API 调用成本

### 实际采用的方案

项目最终采用方案二。

当用户提问时，系统自动从摄像头视频流中连续截取三帧画面，并与用户问题一起发送给 Gemini 进行分析。

该方案在保证视觉理解效果的同时，能够有效降低模型调用成本，因此更适合作为实际运行方案。
